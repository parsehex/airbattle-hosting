# Player Count Notifications

We could totally allow people to subscribe to notifications when the player count on the/a server is high enough, don't need a web extension for that + push notifications _could_ be possible.

This would be part of the landing page and possibly backed by a separate server/backend.

## Follow-up thought

I think doing this in-game 1) might actually work, 2) be more reliable and accurate and appropriate.

This can go in the docs rather than enforcing, but I believe that keeping the game's sound enabled would help even more so that it stays running in the BG, though again, more of a niche feature anyway so shouldn't be more than a tip for users.
