# About Air-Battle

Air-Battle is a community-first continuation of Airmash tooling and hosting.

The short version: I like the game, I want a place where I can host and experiment without stepping on anyone, and I want the project to be open enough that other people can build on it too.

## Where This Came From

I have been tinkering with Airmash forks for years, including smaller gameplay and tooling changes. Over time I started focusing on one core question:

How can this game stay playable and mod-friendly for the long term?

That pushed me toward this repo structure and hosting setup, where the frontend, server, and bots can all evolve together while still being understandable to new contributors.

## What I Am Trying To Build

- Smoother edges.
- A cleaner path for self-hosting and local/offline play.
- Better docs so hosting and modding are less gatekept.
- A practical place to test improvements that can later be shared upstream when it makes sense.

## Why I Work On This

- I enjoy the game itself.
- I want a social and technical outlet.
- The engineering work here helps me build transferable backend, infra, and tooling experience.

## Commitment And Boundaries

This is volunteer work. I will keep maintaining it as long as I still enjoy doing it and have the time.

I cannot promise every requested feature, but I do want to keep communication honest and keep the core server usable.

## Collaborate With Me

If you want to help, I would love collaboration in these areas:

- Server gameplay tuning and balancing.
- Frontend quality-of-life changes.
- Bot behavior and coordination improvements.
- Documentation and setup simplification.
- Testing, deployment, and reliability work.

Best way to reach out right now:

- Open an issue in [parsehex/airbattle-hosting](https://github.com/parsehex/airbattle-hosting) with "collab" in the title.
- Include what area you want to work on and your preferred way to coordinate.

Thanks for taking an interest in the project.
